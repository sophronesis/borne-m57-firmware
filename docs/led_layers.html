the small number in the bottom-right of each key is the LED index (0..57). cells with a dashed border are matrix positions with no LED (PgDn, Mute, MPlay, "}]"). cells with no border are matrix slots that have neither a key nor an LED (corner gaps).

how home row leds work

the home row keys are mod-taps: tap = letter, hold = modifier. the leds on these keys encode which modifiers are currently held using a per-channel color blend.

channel mapping

shift -> red
ctrl -> green
alt -> blue
gui -> intensity bump (base 0x00 -> 0x55, high 0xAA -> 0xFF)

per frame, for each home-row LED:

uint8_t base = gui ? 0x55 : 0x00;
uint8_t high = gui ? 0xFF : 0xAA;
r = shift ? high : base;
g = ctrl  ? high : base;
b = alt   ? high : base;

no mods held -> (0,0,0) = black. that is the "home row off unless you're holding a modifier" baseline. holding shift alone -> dim red. shift+ctrl -> yellow. holding gui adds a constant 0x55 to every channel, so even a bare gui hold lights the row dim grey.

resulting color matrix (16 combinations)

columns are S/C/A/G - 1 = held, 0 = not held. cell is the resulting (r,g,b).

which keys

left half (LEDs 14-18): A S D F G mapped to LGUI_T(A), LALT_T(S), LCTL_T(D), LSFT_T(F), ALL_T(G) (= hyper). the led blend reflects host modifier state, not which physical key is held - a single shift hold from F lights all five home-row leds dim red.

right half (LEDs 43-47): H J K L ; mapped to ALL_T(H), RSFT_T(J), RCTL_T(K), RALT_T(L), RGUI_T(;). mirrors the left side.
